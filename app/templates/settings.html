Token Resolution Order

When making API calls, the token is resolved in this order:

  1. Database — token saved via this settings page {% if token_source == 'db' %}← active{% endif %}
  2. Environment variable — PRISM_TOKEN in your shell or .env file {% if token_source == 'env' %}← active{% endif %}
  3. None — API calls will return 502 errors
